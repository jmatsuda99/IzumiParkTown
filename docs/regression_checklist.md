# 回帰確認チェックリスト

## 目的
コード変更後に、最低限ここだけ確認すれば大きな破綻を見つけやすい、という簡易チェックリストです。

## 事前条件
- 仮想環境または実行環境に `requirements.txt` の依存関係が入っている
- リポジトリ直下に以下の確認用データがある
  - `izumi_power.db`
  - `日射量データ.zip`
  - `test_30min_norm.zip`

## 基本確認

### 1. ランチャー起動
```bash
python launcher.py
```

確認ポイント:
- ツール選択画面が表示される
- `30分値電力解析ツール` と `日射量データ処理ツール` の 2 つが選べる

### 2. 電力解析ツール
ランチャーから `30分値電力解析ツール` を開く。

確認ポイント:
- `最新DBを読込` が実行できる
- `年間概要` が表示される
- `日別負荷` が表示される
- `ヒートマップ` が表示される

### 3. PV 読込と指定日表示
電力解析ツールで `PV正規化CSV/ZIP読込` から `test_30min_norm.zip` を読み込む。

確認ポイント:
- PV 読込後にエラーにならない
- `指定日グラフ` が表示される
- 負荷、疑似PV、受電点の系列が表示される

### 4. 日射量ツール
ランチャーから `日射量データ処理ツール` を開く。

確認ポイント:
- `日射量データ.zip` を選べる
- `一括実行` が完了する
- 抽出、30分補完、正規化の各ログが表示される

## 余力があれば確認
- 電力解析ツールで `CSV出力` を実行できる
- 日射量ツールで `抽出実行`、`補完実行`、`正規化実行` を個別にも実行できる
- `ツール選択へ戻る` でランチャーへ戻れる
